# fanchat

방송인과 팬이 **각자 1:1 채팅방**으로 대화하는 플랫폼. 팬심M처럼 여러 팬의 메시지가 한 방에 뒤섞이는 문제를 해결하기 위해 만들었어요.

## 핵심 아이디어

- **방** = 방송인 1명 ↔ 팬 1명 짝
- 채팅방 목록 화면 하나를 방송인과 팬이 공유해요. 서버가 "내가 낀 방"만 걸러서 보내주므로, 방송인에겐 팬 목록으로, 팬에겐 방송인 목록으로 보여요.
- 팬은 여러 방송인과 각각 방을 가질 수 있어요.
- 방송인이 자기 초대 링크(`/?b=닉네임`)를 뿌리면, 팬이 그 링크로 들어오거나 이름을 입력해 방을 만들어요.

## 기술

- Node.js + Express + ws(WebSocket) — 실시간 통신
- node:sqlite(내장 SQLite) — 대화·계정 영구 저장 (별도 DB 설치 불필요)
- 비밀번호는 scrypt 해시로만 저장 (원문 저장 안 함)

## 내 컴퓨터에서 실행

```bash
npm install
npm start
```

`http://localhost:3100` 접속. 방송인 계정으로 가입 → 초대 링크 복사 → 다른 브라우저(팬)에서 그 링크로 접속.

## 배포 (Render 무료 플랜)

1. 이 저장소를 GitHub에 올림
2. [render.com](https://render.com) → New + → Web Service → 이 저장소 선택
3. Instance Type: **Free**, Start Command: `npm start`
4. Deploy

### 주의: 무료 플랜의 데이터 초기화

Render 무료 플랜은 **재배포할 때마다 서버 파일시스템이 초기화**돼요. `fanchat.db`가 함께 지워지므로 대화·계정이 사라져요. 프로토타입 단계에선 괜찮지만, 실제 운영하려면 영구 저장소(Render Persistent Disk 또는 외부 데이터베이스)를 붙여야 해요.
